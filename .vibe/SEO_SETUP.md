# SEO Setup Guide for Quub Exchange Documentation

## Overview

The Quub Exchange Documentation site now includes comprehensive SEO optimization to improve search engine visibility and ranking.

## What's Included

### 1. Core SEO Configuration (`_config.yml`)

- **Enhanced site metadata**: Title, description, tagline
- **Author information**: Name, email, social profiles
- **Social media integration**: Twitter, OpenGraph, LinkedIn
- **Keywords**: Relevant industry keywords for better discoverability
- **Plugins**: jekyll-seo-tag, jekyll-sitemap, jekyll-feed

### 2. Meta Tags (in `_layouts/default.html`)

- **Primary meta tags**: title, description, keywords, author
- **Open Graph tags**: For Facebook and LinkedIn sharing
- **Twitter Card tags**: For Twitter sharing with rich previews
- **Canonical URLs**: To prevent duplicate content issues
- **Schema.org JSON-LD**: Structured data for search engines

### 3. Additional SEO Components (`_includes/seo-tags.html`)

- **Breadcrumb schema**: For better navigation in search results
- **TechArticle schema**: For API documentation pages
- **RSS feed**: For content syndication
- **Preconnect directives**: For faster page loading
- **Theme colors**: For mobile browsers

### 4. Robots.txt

- Allows all search engine crawlers
- References sitemap.xml
- Blocks non-public directories

### 5. Sitemap

- Auto-generated by jekyll-sitemap plugin
- Lists all pages for search engine indexing
- Located at `/sitemap.xml`

## Setup Instructions

### Step 1: Update Configuration

Edit `_config.yml` and update the following:

```yaml
# Update with your actual social media handles
twitter:
  username: your_actual_twitter_handle

# Add your Google Search Console verification code
google_site_verification: YOUR_VERIFICATION_CODE_HERE

# Update author email
author:
  email: your-email@quub.fi
```

### Step 2: Add Social Images

Create and add the following images:

1. **Logo**: `/assets/images/quub-logo.png`

   - Size: 512x512px or larger
   - Square format for best compatibility

2. **OpenGraph Image**: `/assets/images/quub-og-image.png`
   - Size: 1200x630px (Facebook/LinkedIn recommended)
   - Should include branding and tagline

### Step 3: Google Search Console Setup

1. Go to [Google Search Console](https://search.google.com/search-console)
2. Add your property: `https://quub-fi.github.io/quub-exchange-docs/`
3. Verify ownership using the verification code
4. Add the code to `_config.yml` under `google_site_verification`
5. Submit your sitemap: `https://quub-fi.github.io/quub-exchange-docs/sitemap.xml`

### Step 4: Submit to Search Engines

#### Google

- Submit sitemap in [Google Search Console](https://search.google.com/search-console)
- URL: `https://quub-fi.github.io/quub-exchange-docs/sitemap.xml`

#### Bing

- Submit sitemap in [Bing Webmaster Tools](https://www.bing.com/webmasters)
- URL: `https://quub-fi.github.io/quub-exchange-docs/sitemap.xml`

### Step 5: Verify Implementation

Test your SEO implementation:

1. **Google Rich Results Test**

   - https://search.google.com/test/rich-results
   - Test any page URL to verify structured data

2. **Facebook Sharing Debugger**

   - https://developers.facebook.com/tools/debug/
   - Test how pages appear when shared on Facebook

3. **Twitter Card Validator**

   - https://cards-dev.twitter.com/validator
   - Test how pages appear when shared on Twitter

4. **Schema Markup Validator**
   - https://validator.schema.org/
   - Validate JSON-LD structured data

## Page-Level SEO

For individual pages, add front matter:

```yaml
---
layout: docs
title: Your Page Title
description: A compelling 150-160 character description of the page content
keywords: keyword1, keyword2, keyword3
image: /assets/images/page-specific-image.png
---
```

## Best Practices

### Content Optimization

1. **Title Tags**: Keep under 60 characters
2. **Meta Descriptions**: 150-160 characters, include CTA
3. **Headings**: Use H1 for page title, H2-H6 for structure
4. **Keywords**: Natural placement in first paragraph
5. **Internal Linking**: Link related documentation pages
6. **Alt Text**: Describe all images for accessibility and SEO

### Performance

- Images optimized (compressed, proper format)
- Minimize CSS/JS file sizes
- Use CDN for external resources
- Enable caching

### Mobile Optimization

- Responsive design (already implemented)
- Touch-friendly navigation
- Fast loading times
- Proper viewport settings

## Monitoring & Analytics

### Recommended Tools

1. **Google Analytics** (add tracking code to `_includes/analytics.html`)
2. **Google Search Console** (monitor search performance)
3. **Bing Webmaster Tools** (Bing search visibility)
4. **Ahrefs/SEMrush** (comprehensive SEO analysis)

### Key Metrics to Track

- Organic search traffic
- Keyword rankings
- Click-through rate (CTR)
- Bounce rate
- Page load time
- Mobile usability
- Index coverage
- Backlinks

## Troubleshooting

### Pages Not Indexed

1. Check robots.txt isn't blocking
2. Verify sitemap is accessible
3. Submit URL for indexing in Search Console
4. Ensure proper internal linking

### Poor Rankings

1. Improve content quality and depth
2. Add more relevant keywords naturally
3. Build backlinks from reputable sources
4. Improve page load speed
5. Enhance mobile experience

### Duplicate Content

- Canonical URLs are set automatically
- Ensure no duplicate pages exist
- Use 301 redirects for moved content
